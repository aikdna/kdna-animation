# Frontend Motion System

Domain: `frontend_motion_system`

Encodes **design system engineering judgment** for motion — how teams turn one-off animations into tokenized, reusable, reviewable primitives.

This domain covers:
- **Tokens before effects**: Duration scales, easing curves, and distance tokens make motion repeatable
- **Primitives over one-offs**: Shared animation behaviors prevent copy-paste drift and enable design review
- **State transitions need semantics**: Name motion by intent (modal-enter), not implementation (scale-fade-400ms)
- **Motion docs must be executable**: Documentation must include copy-pasteable code and live previews

Part of the open-source [Animation KDNA Cluster](../README.md).
